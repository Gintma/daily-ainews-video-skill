# Codex News Video Workflow

This folder is the standalone workspace for generating news videos with Codex.

## Goal

Use Codex as the director and automation agent:

1. Read `assets/chinese-vertical-video-style-prompt.md` for the visual/editorial style.
2. Search the web for news for the requested topic, time window, and item count.
3. Verify dates, sources, and metrics.
4. Translate and edit the stories into Chinese.
5. Write a structured `episode.json`.
6. Decide per story whether to generate an abstract editorial image or use a
   pure typography scene.
7. Let the LLM generate story-specific `compositions/story-XX.html` files.
8. Use scripts to assemble the HyperFrames shell, audio, captions, and render.
9. Use local VoxCPM for voice generation with a stable reference voice.
10. Stitch the voice track.
11. Render the final 9:16 MP4.

## Hard Rules

- Each run gets a new output directory under `daily-news-video-runs/`.
- Default run mode is interactive. Stop for user confirmation at key
  checkpoints unless the user explicitly asks for fully automatic mode.
- Story count is dynamic. Do not hard-code five stories.
- Duration is dynamic. Do not force 60 seconds unless requested.
- `episode.json` controls the project. Story HTML is designed by the LLM from
  the episode data. Shell files, scripts, captions, and audio wiring are
  generated by deterministic scripts.
- Use reliable sources and include source URLs in `episode.json`.
- Do not show full URLs in the rendered video. Use publisher/date source badges
  on screen and keep full URLs in `episode.json`.
- Every story must include `visual_direction.image_strategy`.
- Generate AI images only for stories whose `image_strategy` is `generate`;
  do not force images into every story.
- Use `assets/voice-reference.wav` and `assets/voice-reference-transcript.txt` as the default VoxCPM voice
  reference unless the user provides a different voice profile.

## Interactive Checkpoints

In interactive mode, stop and wait for user approval at these points:

1. News selection
   - Show candidate stories with Chinese title, source, date, summary, and
     reason for inclusion.
   - Ask whether to keep, replace, reorder, or change the count.

2. Chinese script and translation
   - Show `headline_lines`, `metric`, `callout`, and narration script.
   - Apply user edits before writing or updating `episode.json`.
   - Once approved, continue automatically through visual planning, optional
     image generation, story HTML, voice, stitching, render, and verification.

If the user says "继续", that approves only the current checkpoint.

## Episode Contract

`episode.json` must contain:

- `meta`: title, slug, language, created date, optional publishing context
- `timing`: cover seconds, story seconds, outro seconds, total seconds
- `cover`: screen text and narration
- `stories`: dynamic list of story objects
- story visual direction: layout, image strategy, composition, shapes, image
  prompt, motion
- conditional story visuals: `visual_prompt`, `image_path` only when
  `image_strategy` is `generate`
- `outro`: screen text and narration
- `sources`: all URLs used

The number of generated `story-XX.html` files must equal
`episode.stories.length`.

## Codex Run Steps

0. Setup check
   - Run:
     ```bash
     bash scripts/setup_check.sh
     ```
   - If dependencies are missing, choose the highest available mode:
     script/project only, video without local voice, or full local render.

1. Research
   - Search the web.
   - Prefer official sources and primary announcements.
   - Use media roundups only for discovery or popularity signals.
   - Verify each story date against the requested time window.
   - Stop for the news selection checkpoint in interactive mode.

2. Write `episode.json`
   - Translate into Chinese.
   - Keep narration compact and natural.
   - Give every story a metric or punchy keyword.
   - Add `visual_direction` for each story, including `layout`,
     `image_strategy`, `composition`, `shapes`, `image_prompt`, and `motion`.
   - Use `image_strategy: "generate"` only when a generated abstract image adds
     meaning. Use `image_strategy: "none"` when typography, numbers, source
     badges, color, and motion are clearer.
   - When `image_strategy` is `none`, do not add decorative diagrams,
     pseudo-SVG motifs, node-link graphics, icon clusters, or abstract
     illustration panels.
   - Keep source URLs in data only; use short publisher/date badges on screen.
   - Stop for the Chinese script checkpoint in interactive mode. After approval,
     continue automatically through the rest of the workflow.

3. Generate selected images
   - For stories with `image_strategy: "generate"`, generate abstract editorial
     images and save them to `media/images/story-XX.png`.
   - Set `story.image_path` for generated stories.
   - Do not scrape or download publisher images unless explicitly requested.

4. Generate LLM-designed story HTML
   - Read `assets/chinese-vertical-video-style-prompt.md`.
   - Write one file per story:
     `daily-news-video-runs/<slug>/compositions/story-XX.html`.
   - Each file must be a standalone HyperFrames composition with a matching
     composition id, `1080x1920` dimensions, `data-duration`, and a paused GSAP
     timeline assigned to `window.__timelines["story-XX"]`.
   - The LLM should design layout and animation per story. Do not reuse the
     same layout for every story unless the content genuinely calls for it.
   - Keep publisher/date source badges on screen; keep full URLs in data only.

5. Generate HyperFrames project shell
   - Run:
     ```bash
     python3 scripts/generate_hyperframes_project.py daily-news-video-runs/<slug>/episode.json
     ```
   - Existing `story-XX.html` files are preserved by default. The script only
     updates their `data-duration`.
   - Use `--overwrite-stories` only to fall back to the deterministic template.
   - If `assets/background-music.mp3` exists, the generator copies it to
     `media/background-music.mp3` and adds it as a low-volume BGM track.
   - The voice track is added only after `media/voice.mp3` exists.

6. Generate voice with VoxCPM
   - Run:
     ```bash
     bash scripts/generate_voice_voxcpm.sh daily-news-video-runs/<slug>
     ```

7. Stitch audio
   - Run:
     ```bash
     python3 scripts/stitch_audio.py daily-news-video-runs/<slug>
     ```
   - This writes `media/voice.mp3` and updates `index.html` with the voiceover
     audio track.
   - It also reads the real duration of every VoxCPM clip and writes
     `timing.scene_seconds` plus `timing.total_seconds` back to `episode.json`.
   - Do not use speed-up to force narration into fixed slots. Scene durations
     follow the audio.

8. Regenerate HyperFrames project shell after stitching audio
   - Run:
     ```bash
     python3 scripts/generate_hyperframes_project.py daily-news-video-runs/<slug>/episode.json
     ```
   - This updates scene timings, captions, BGM duration, and voiceover duration
     from the audio-driven `episode.json`.
   - Existing LLM-designed `story-XX.html` files are preserved.

9. Render
   - Run:
     ```bash
     bash scripts/render_hyperframes.sh daily-news-video-runs/<slug>
     ```

## Notes

- This skill should degrade gracefully. If HyperFrames, VoxCPM, or ffmpeg are
  missing, still produce the research, Chinese scripts, and `episode.json`.
- VoxCPM `design` mode may change voice between clips. Use `clone` mode with
  `voice-reference.wav` and `voice-reference-transcript.txt` for stable voice.
- If a clip exceeds its scene slot, `stitch_audio.py` may apply mild `atempo`.
  Prefer revising narration if the required speed-up is above `1.30`.
- Captions can start as estimated timings from the scene slots. For production,
  transcribe `media/voice.mp3` and regenerate word-level captions.
- Captions should use kinetic word highlighting by default. Brand names,
  numbers, and metric terms should get accent color.
